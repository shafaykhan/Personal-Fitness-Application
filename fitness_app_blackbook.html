CHAPTER 1: INTRODUCTION

1.1 Introduction to the Project

The rapid advancement of digital technology has transformed almost every aspect of human life, including health and fitness. In the post-pandemic world, individuals have become increasingly aware of the importance of maintaining a healthy lifestyle. The Personal Fitness Application is a specialized web platform designed to bridge the gap between busy schedules and fitness goals.

1.2 Motivation and Need of the System

The primary motivation behind this project is the observation that while many people start fitness routines with high enthusiasm, a significant percentage drop out due to a lack of visible progress or an organized tracking mechanism.

  • Information Silos: Most users use separate tools for diet, steps, and weight tracking.
  • Lack of Accountability: Without a historical record, users find it difficult to stay motivated.
  • Data Security: Our system prioritizes private hosting and token-based security.

1.3 Objectives of the Project

  1. Centralization: Provide a single source of truth for all fitness data.
  2. Security: Implement JWT-based stateless authentication.
  3. Visualization: Develop a dynamic dashboard with progress bars.
  4. Usability: Create an intuitive interface using AdminLTE.

1.4 Scope of the Project

The scope includes user registration, secure login, dashboard insights, and full CRUD operations for Workouts, Diet, and Health vitals. It ensures accessibility via desktop and mobile browsers.

CHAPTER 2: LITERATURE REVIEW

2.1 Existing Systems

Current solutions range from manual notebooks to cluttered commercial apps like MyFitnessPal. Manual logs lack calculation, while commercial apps often sell user data or require expensive subscriptions.

2.2 Proposed System

The proposed system focuses on Simplicity and Security. By using Spring Security, we ensure that data is private. The UI is designed to provide a professional "Executive Dashboard" feel.

CHAPTER 3: SYSTEM ANALYSIS

3.1 Software Requirements

  • Backend: Spring Boot 3.x
  • Database: MySQL 8.0
  • Frontend: HTML5, JavaScript, Bootstrap 5
  • UI Theme: AdminLTE

3.2 Functional Requirements

  • Identity Module: Login/Registration via JWT.
  • Workout Module: Logging duration, type, and calories.
  • Goal Module: Target setting and progress bars.

CHAPTER 4: SYSTEM DESIGN

4.1 System Architecture

The application follows an N-Tier Architecture consisting of the Client Layer, API Layer (REST Controllers), Business Logic Layer (Services), and Persistence Layer (JPA/MySQL).

4.2 Security Architecture (JWT)

The system uses stateless authentication where a token is issued upon login and stored in the browser's localStorage. This token is sent in the header of every subsequent API request.

CHAPTER 5: IMPLEMENTATION

Implementation involved developing Spring Boot RESTful services and a dynamic frontend. The api.js utility was created to handle global context paths and token injection seamlessly across all modules.
